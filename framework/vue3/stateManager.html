状态管理

  • vue 中常用 vuex、pinia 做状态管理工作。
  • 从项目级的角度看，状态管理就是单例模式。
  • 现在（2025年）不需要状态管理的情况太少了。

vuex已经落伍了

pinia

自制状态管理

利用 vue3 中 reactive 做状态管理。 好像单例模式。

import { createApp, reactive } from 'vue'
const obj = reactive({
  msg: 'string',
})
const appA = createApp({
  data() {
    return obj
  },
}).mount('#a')
const appB = createApp({
  data() {
    return obj
  },
}).mount('#b')

import {reactive} from 'vue'
export let store = {
    debug: true,
    state: reactive({
        msg: 'str'
    }),
    setMsgAction(nv) {
        if (this.debug) {
            clog('...')
        }
        this.state.msg = nv
    }
    clearMsgAction() {
        if (this.debug) {
            clog('...')
        }
        this.state.msg = ''
    }
}
// 然后在组件中引入并使用。

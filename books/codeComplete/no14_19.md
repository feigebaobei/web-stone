# no.14 直线型代码的组织

面向顺序编程
顺序执行的方式：

- 组织代码，使依赖关系显而易见。
- 子程序的命名要揭示依赖关系。
- 使用子程序参数来揭示依赖关系。

自上而下的顺序对可读性贡献最大。  
相关语句分在一组。

# no.15 使用条件语句

```
if else
switch case
```

先写正常情况，再写否正常情况。
正常情况的逻辑写在 if 内（非 else 内）。  
用布尔函数调用可简化复杂测试  
最常见的情况放在最前面。

选择最有效的 case 顺序

- 按频率排列
- 按字母、数字顺序排列
- 正常情况定在最前面
- default 检测错误
